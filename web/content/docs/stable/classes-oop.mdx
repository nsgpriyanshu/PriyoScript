## What is OOP?

Object-Oriented Programming (OOP) models software as objects (state + behavior). Common goals:

- Encapsulation: keep related data and behavior together
- Reuse: avoid duplicating logic
- Extensibility: add new behavior by inheritance/composition

## Example 1: Basic class

```priyo
monalisa {
  lisaaFamily Student {
    lisaaTask init(name) {
      priyoSelf.name = name
    }

    lisaaTask intro() {
      priyoTell("I am " + priyoSelf.name)
    }
  }

  priyoKeep s = priyoCreate Student("Priyo")
  s.intro()
}
```

## Example 2: Inheritance

```priyo
monalisa {
  lisaaFamily Person {
    lisaaTask init(name) {
      priyoSelf.name = name
    }
  }

  lisaaFamily Teacher lisaaInherit Person {
    lisaaTask init(name, subject) {
      priyoParent(name)
      priyoSelf.subject = subject
    }
  }
}
```

## Example 3: Static members

```priyo
monalisa {
  lisaaFamily MathUtil {
    lisaaStable priyoPromise PI = 3.14159

    lisaaStable lisaaTask square(n) {
      priyoGiveBack n * n
    }
  }

  priyoTell(MathUtil.square(4))
}
```

## Example 4: Domain-style object model

```priyo
monalisa {
  lisaaFamily Course {
    lisaaTask init(title, credits) {
      priyoSelf.title = title
      priyoSelf.credits = credits
    }
  }

  priyoKeep c = priyoCreate Course("Compiler Design", 4)
  priyoTell(c.title)
}
```

Language mapping:

- Java: closest conceptual match (classes + constructors + inheritance)
- JavaScript: class syntax exists but prototype-based underneath
- Python: class model with explicit `self`
- C: not OOP by default (struct + function patterns)

<Callout title="Constructor rule" type="error">
  In child `init`, `priyoParent(...)` must be called first and only once.
</Callout>
