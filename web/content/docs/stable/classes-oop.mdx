## What is OOP?

OOP (Object-Oriented Programming) means grouping data and behavior together in objects.

Core ideas:

- Object = state + behavior
- Class = blueprint for objects
- Inheritance = reuse parent behavior

## Example 1: Basic class

```priyo
monalisa {
  lisaaFamily Student {
    lisaaTask init(name) {
      priyoSelf.name = name
    }
    lisaaTask intro() {
      priyoTell("I am " + priyoSelf.name)
    }
  }
  priyoKeep s = priyoCreate Student("Priyo")
  s.intro()
}
```

## Example 2: Inheritance

```priyo
monalisa {
  lisaaFamily Person {
    lisaaTask init(name) {
      priyoSelf.name = name
    }
  }
  lisaaFamily Teacher lisaaInherit Person {
    lisaaTask init(name, subject) {
      priyoParent(name)
      priyoSelf.subject = subject
    }
  }
}
```

## Example 3: Static members

```priyo
monalisa {
  lisaaFamily MathUtil {
    lisaaStable lisaaTask square(n) {
      priyoGiveBack n * n
    }
  }
  priyoTell(MathUtil.square(4))
}
```

## Example 4: Real model

```priyo
monalisa {
  lisaaFamily Course {
    lisaaTask init(title, credits) {
      priyoSelf.title = title
      priyoSelf.credits = credits
    }
  }
  priyoKeep c = priyoCreate Course("Compiler", 4)
  priyoTell(c.title)
}
```

## How execution works (like a child)

Class is cookie cutter. Object is cookie.

- `lisaaFamily` makes cutter
- `priyoCreate` makes cookie
- `init` puts details in cookie
- methods read/use those details

Comparison:

- Java: closest class model
- JS: class syntax, prototype internals
- Python: class with `self`
- C: manual struct + functions

<Callout title="Constructor rule" type="error">
  In child `init`, `priyoParent(...)` must be first and only once.
</Callout>
