## Theory

Errors happen when program assumptions are broken.

Good error handling means:

- Detect early
- Explain clearly
- Recover when possible

## Example 1: try/catch/finally

```priyo
monalisa {
  prakritiTry {
    prakritiThrow "Something failed"
  } prakritiCatch (err) {
    priyoTell.Error("Caught: " + err.message)
  } prakritiAtEnd {
    priyoTell.Info("Cleanup done")
  }
}
```

## Example 2: Validation throw

```priyo
monalisa {
  lisaaTask divide(a, b) {
    prakritiIf (b == 0) {
      prakritiThrow "Division by zero is not allowed"
    }
    priyoGiveBack a / b
  }
}
```

## Example 3: User-friendly flow

```priyo
monalisa {
  prakritiTry {
    prakritiThrow "Invalid input"
  } prakritiCatch (err) {
    priyoTell.Warn(err.message)
  }
}
```

## How execution works (like a child)

Imagine carrying water in a glass:

- `try` = carry carefully
- if falls, `catch` = clean spill
- `finally` = always wash hands
