## 1. Theory

Errors are normal in programming.  
Good software does not ignore them; it handles them clearly.  
PriyoScript gives structured error handling keywords.  
`prakritiTry` protects risky code.  
`prakritiCatch` handles failures.  
`prakritiAtEnd` runs cleanup code always.  
`prakritiThrow` creates explicit custom failures.  
Diagnostics include code, stage, location, and tip for fast recovery.  
This design helps beginners understand both mistake and fix.

## 2. Program Examples

### Example 1: Basic try/catch/finally

```priyo
monalisa {
  prakritiTry {
    prakritiThrow "Something failed"
  } prakritiCatch (err) {
    priyoTell.Error("Caught: " + err.message)
  } prakritiAtEnd {
    priyoTell.Info("Cleanup done")
  }
}
```

Output:

```text
Caught: Something failed
Cleanup done
```

### Example 2: Validate Input Before Operation

```priyo
monalisa {
  lisaaTask divide(a, b) {
    prakritiIf (b == 0) {
      prakritiThrow "Division by zero is not allowed"
    }
    priyoGiveBack a / b
  }

  prakritiTry {
    priyoTell(divide(10, 0))
  } prakritiCatch (err) {
    priyoTell.Warn(err.message)
  }
}
```

Output:

```text
Division by zero is not allowed
```

### Example 3: Manual Business Rule Error

```priyo
monalisa {
  lisaaTask enroll(age) {
    prakritiIf (age < 18) {
      prakritiThrow "Minimum age is 18"
    }
    priyoGiveBack "Enrolled"
  }

  prakritiTry {
    priyoTell(enroll(16))
  } prakritiCatch (err) {
    priyoTell.Error(err.message)
  }
}
```

Output:

```text
Minimum age is 18
```

### Example 4: Continue Program After Catch

```priyo
monalisa {
  prakritiTry {
    prakritiThrow "Network timeout"
  } prakritiCatch (err) {
    priyoTell.Warn("Retry later")
  }

  priyoTell("Program still running")
}
```

Output:

```text
Retry later
Program still running
```

### Example 5: Inspect Error Metadata

```priyo
monalisa {
  prakritiTry {
    prakritiThrow "Custom failure"
  } prakritiCatch (err) {
    priyoTell(err.message)
    priyoTell(err.code)
  }
}
```

Output:

```text
Custom failure
(implementation-specific code)
```

## 3. Execution Flow

1. Code inside `prakritiTry` starts.
2. If no error occurs, `catch` is skipped.
3. If error occurs, control jumps to `prakritiCatch`.
4. `err` object carries details like message/code.
5. `prakritiAtEnd` runs whether error happened or not.
6. Program continues after handling unless explicitly stopped.

## 4. Comparison With Other Languages

Java:

```java
try {
  risky();
} catch (Exception e) {
  System.out.println(e.getMessage());
} finally {
  cleanup();
}
```

JavaScript:

```javascript
try {
  risky()
} catch (e) {
  console.log(e.message)
} finally {
  cleanup()
}
```

Python:

```python
try:
    risky()
except Exception as e:
    print(str(e))
finally:
    cleanup()
```

C:

- C has no native `try/catch`; errors are handled via return codes, `errno`, or `goto` cleanup patterns.

PriyoScript:

```priyo
prakritiTry { ... } prakritiCatch (err) { ... } prakritiAtEnd { ... }
```

Difference summary:

- PriyoScript, Java, JS, Python use structured exception flow.
- C uses manual error handling patterns.
