## Theory

Error handling prevents silent failures and makes programs predictable in failure scenarios.

PriyoScript supports structured exception flow:

- `prakritiTry`
- `prakritiCatch`
- `prakritiAtEnd`
- `prakritiThrow`

## Example 1: try/catch/finally

```priyo
monalisa {
  prakritiTry {
    prakritiThrow "Something failed"
  } prakritiCatch (err) {
    priyoTell.Error("Caught: " + err.message)
  } prakritiAtEnd {
    priyoTell.Info("Cleanup done")
  }
}
```

## Example 2: Validation + throw

```priyo
monalisa {
  lisaaTask divide(a, b) {
    prakritiIf (b == 0) {
      prakritiThrow "Division by zero is not allowed"
    }
    priyoGiveBack a / b
  }

  priyoTell(divide(10, 2))
}
```

## Example 3: User-friendly domain checks

```priyo
monalisa {
  lisaaTask validateAge(age) {
    prakritiIf (age < 0) {
      prakritiThrow "Age cannot be negative"
    }
  }

  prakritiTry {
    validateAge(-1)
  } prakritiCatch (err) {
    priyoTell.Warn(err.message)
  }
}
```

## Reading diagnostics

Each diagnostic includes:

- Error code (example: `PSYN-002`)
- Stage (`syntax`, `compile`, `runtime`, or `engine`)
- File location and source span
- Human-readable tip
- Docs link

```text
Code: PSYN-002
Stage: syntax
Location: file.priyo:3:10
Source: prakritiIff (x > 0) {
Span:          ^^^^^^^^^^
Tip: Use `prakritiIf` for conditional branches.
Docs: https://priyyoscript.vercel.app/docs/stable/errors-reference#psyn-002
```

<Callout title="Engine vs user errors" type="error">
  Engine errors usually indicate language implementation issues. Syntax/compile/runtime errors are
  generally user-code mistakes.
</Callout>
