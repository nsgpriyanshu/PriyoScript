# Error Handling

## Try / catch / finally

```priyo
monalisa {
  prakritiTry {
    prakritiThrow "Something failed"
  } prakritiCatch (err) {
    priyoTell.Error("Caught: " + err.message)
  } prakritiAtEnd {
    priyoTell.Info("Cleanup done")
  }
}
```

## Throw custom errors

```priyo
monalisa {
  lisaaTask divide(a, b) {
    prakritiIf (b == 0) {
      prakritiThrow "Division by zero is not allowed"
    }

    priyoGiveBack a / b
  }

  priyoTell(divide(10, 2))
}
```

## Reading diagnostics

PriyoScript diagnostics include:

- Error code (example: `PSYN-002`)
- Stage (`syntax`, `compile`, `runtime`, or internal engine)
- File location and source span
- Human-readable message and tip
- Docs link for the specific error code

```text
Code: PSYN-002
Stage: syntax
Location: file.priyo:3:10
Source: prakritiIff (x > 0) {
Span:          ^^^^^^^^^^
Tip: Use `prakritiIf` for conditional branches.
Docs: https://priyyoscript.vercel.app/docs/stable/errors-reference#psyn-002
```

<Callout title="Engine vs user errors" type="error">
  Engine/internal errors point to language implementation issues. Syntax/compile/runtime errors
  usually come from user code.
</Callout>
