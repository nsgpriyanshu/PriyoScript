## Theory

Arrays store ordered collections. PriyoScript arrays are first-class values and support practical operations for reading, transforming, and iterating.

## Example 1: Create, read, update

```priyo
monalisa {
  priyoKeep nums = [10, 20, 30]
  priyoTell(nums[0])
  nums[1] = 25
  priyoTell(nums)
}
```

## Example 2: Slicing

```priyo
monalisa {
  priyoKeep nums = [10, 20, 30, 40]
  priyoTell(nums[0:2])
  priyoTell(nums[:2])
  priyoTell(nums[1:])
}
```

## Example 3: Destructuring with defaults

```priyo
monalisa {
  priyoKeep values = [7, 8]
  priyoChange[(first, second, third = 99)] = values
  priyoTell(first)
  priyoTell(second)
  priyoTell(third)
}
```

## Example 4: `priyoArray` helpers

```priyo
monalisa {
  lisaaTask isEven(x) {
    priyoGiveBack x % 2 == 0
  }

  priyoKeep values = [1, 2, 3, 4, 5]
  priyoKeep evens = priyoArray.filter(values, isEven)
  priyoTell(priyoArray.length(evens))
}
```

Comparison:

- C: fixed-size arrays, manual memory patterns
- Java: arrays plus rich collection frameworks
- JavaScript/Python: dynamic arrays/lists with high-level helpers

<Callout title="Recommendation" type="idea">
  Prefer `priyoArray` callbacks for transformation logic instead of manual index loops.
</Callout>
