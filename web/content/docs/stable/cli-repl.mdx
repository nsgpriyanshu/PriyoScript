## 1. Theory

CLI means Command Line Interface: run full files from terminal.  
REPL means Read-Eval-Print Loop: run one snippet at a time interactively.  
Use CLI for repeatable project execution.  
Use REPL for fast experiments and quick checks.  
Trace mode helps you debug execution flow instruction-by-instruction.  
You can also place `priyoBreak("label")` in code to mark debug checkpoints.  
Both are important in daily development.  
PriyoScript provides commands for help, syntax, errors, and explanations.  
REPL keeps runtime memory between lines, which is useful for testing ideas.  
For production scripts, prefer CLI-based `.priyo` files in version control.

## 2. Program Examples

### Example 1: Show Help

```bash
monalisa -h
```

Output (short):

```text
PriyoScript CLI help...
```

### Example 2: Run a File

```bash
monalisa hello.priyo
```

Output:

```text
Hello, PriyoScript!
```

### Example 3: Start REPL

```bash
monalisa -repl
```

Output (short):

```text
Hey, this is PriyoScript REPL...
priyo>
```

### Example 4: REPL Session

```text
priyo> priyoKeep x = 12
priyo> priyoTell(x * 2)
24
```

Output:

```text
24
```

### Example 5: Error Discovery Commands

```bash
monalisa -errors
monalisa -explain PSYN-002
```

Output (short):

```text
List of error codes...
Explanation for PSYN-002...
```

### Example 6: Trace Mode

```bash
monalisa -trace examples/basics/for-loop.priyo
```

Output (short):

```text
[TRACE #1] file=examples/basics/for-loop.priyo frame=main op=...
[TRACE #2] file=examples/basics/for-loop.priyo frame=main op=...
...
```

### Example 7: Breakpoint-Style Hook

```priyo
monalisa {
  priyoKeep total = 10
  priyoBreak("after-total")
  priyoTell(total)
}
```

Output (short):

```text
[BREAK #1] after-total
10
```

## 3. Execution Flow

CLI flow:

1. You pass file path.
2. CLI loads full source.
3. Parser/compiler/VM run once.
4. Program exits with output or error.

REPL flow:

1. You type one line/snippet.
2. REPL evaluates it immediately.
3. State stays in memory for next line.
4. You repeat until `.exit`.

Trace flow:

1. You run with `-trace`.
2. VM emits execution lines for each instruction.
3. If `priyoBreak(...)` exists, VM prints a breakpoint marker.
4. On runtime error, PriyoScript includes source-level stack context.

## 4. Comparison With Other Languages

Node.js CLI/REPL:

- `node file.js`
- `node` (interactive REPL)

Python CLI/REPL:

- `python script.py`
- `python` (interactive shell)

Java:

- Compile + run model (`javac`, `java`) and optional JShell for REPL-like experience.

C:

- Compile then execute binary (`gcc`, `./a.out`), no default built-in REPL.

PriyoScript:

- `monalisa file.priyo`
- `monalisa -repl`
- `monalisa -trace file.priyo`

Difference summary:

- PriyoScript behaves like modern interpreted workflows (Node/Python) with dedicated language diagnostics.

<Callout title="Limitations" type="warn">
  REPL is ideal for learning and prototyping, but long programs should be stored in `.priyo` files
  and executed via CLI.
</Callout>

<Callout title="Debug Tip" type="idea">
  Start with normal run. If behavior is unclear, rerun with `-trace` and place
  `priyoBreak("checkpoint")` at important points.
</Callout>
