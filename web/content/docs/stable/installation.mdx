## Overview

PriyoScript supports two stable installation paths for end users:

1. npm global install (cross-platform)
2. Standalone Windows installer bundle (no npm required on target machine)

Both paths install the `monalisa` command so you can run `.priyo` files from any folder.

<Callout title="Recommended choice" type="idea">
If you already have Node.js installed, use npm global install.  
If your machine has no Node.js or npm, use the Windows standalone installer bundle.
</Callout>

---

## 1. Prerequisites

### For npm install
- Node.js 18 or higher
- npm (comes with Node.js)

### For standalone Windows install
- Windows PowerShell
- Installer bundle downloaded from PriyoScript releases

---

## 2. What Happens During Installation

When you install PriyoScript:

1. The PriyoScript CLI is installed on your system.
2. Command shims are created (`monalisa`).
3. Your PATH is updated so the command works from any folder.
4. You verify installation using `monalisa -h`.

---

## 3. Method A: npm Global Install (All Platforms)

### Install

```bash
npm install -g priyoscript
```

### Verify Installation

```bash
monalisa -h
monalisa -repl
```

### Run Your First Program

Create a file named `hello.priyo`:

```priyo
monalisa {
  priyoTell("Hello, PriyoScript!")
}
```

Run it:

```bash
monalisa hello.priyo
```

Expected output:

```text
Hello, PriyoScript!
```

---

### Update PriyoScript

```bash
npm install -g priyoscript@latest
```

---

### Uninstall (npm method)

```bash
npm uninstall -g priyoscript
```

---

## 4. Method B: Standalone Windows Installer (No npm Required)

This method is ideal if the system does not have Node.js or npm installed.

### 4.1 Download Installer

Download the release artifact:

* `priyoscript-installer.zip`

From:

* PriyoScript GitHub Releases

---

### 4.2 Run Setup

Extract the zip file.

Open PowerShell inside the extracted folder and run:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup-priyoscript.ps1 -Scope User -Version <version>
```

For machine-wide installation (Admin PowerShell):

```powershell
powershell -ExecutionPolicy Bypass -File .\setup-priyoscript.ps1 -Scope Machine -Version <version>
```

---

### 4.3 What Setup Configures Automatically

* Installs `priyoscript.exe`
* Creates command shims:

  * `monalisa.cmd`
* Adds installation `bin` folder to PATH
* Registers uninstall entry
* Stores installation metadata

---

### 4.4 Verify Installation

Close the terminal.
Open a new terminal and run:

```bash
monalisa -h
monalisa -repl
```

---

### 4.5 Uninstall Standalone Install

User install:

```powershell
powershell -ExecutionPolicy Bypass -File .\uninstall-priyoscript.ps1 -Scope User
```

Machine install:

```powershell
powershell -ExecutionPolicy Bypass -File .\uninstall-priyoscript.ps1 -Scope Machine
```

---

## 5. Troubleshooting

<Callout title="Command not found" type="error">
If `monalisa` is not recognized:

1. Close the terminal completely.
2. Open a new terminal.
3. Run `monalisa -h` again.

If it still fails, check whether the install `bin` folder exists in your system PATH. 
</Callout>

<Callout title="Execution policy error" type="warn">
If PowerShell blocks setup scripts, make sure you run the command exactly as shown with `-ExecutionPolicy Bypass`.
</Callout>
