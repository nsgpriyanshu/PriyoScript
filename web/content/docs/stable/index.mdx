PriyoScript is a human-first interpreted language with readable keywords, bytecode VM execution, first-class arrays, modules, OOP, and structured diagnostics.

<Callout title="Stable Scope" type="warn">
  This handbook documents stable features only. Canary docs may include incomplete or changing
  behavior.
</Callout>

## History of PriyoScript

PriyoScript started as a language-learning and language-design project focused on one core question:

**Can programming syntax feel closer to natural human reading while still remaining precise for real software?**

The project evolved from a small interpreter prototype into a full pipeline with lexer, parser, compiler, VM, CLI, REPL, modules, OOP, arrays, and structured error diagnostics.

## Why PriyoScript was made

PriyoScript was created to reduce fear and friction for beginners while still staying useful for serious developers.

Key motivation:

- Make language constructs feel readable and intention-driven
- Keep errors humanized so users can recover fast
- Provide a predictable runtime model for real project workflows
- Bridge learning from common ecosystems (JavaScript/Python/C/Java)

<Callout title="Design principle" type="idea">
  PriyoScript aims to be as close to plain-English intent as possible without sacrificing
  compiler/runtime clarity.
</Callout>

## First runnable program

```priyo title="hello.priyo"
monalisa {
  priyoTell("Hello, PriyoScript!")
}
```

```bash
monalisa hello.priyo
```

## PriyoScript at a glance

- Entry block: `monalisa { ... }`
- Variables: `priyoKeep`, `priyoChange`, `priyoPromise`
- Functions: `lisaaTask ... priyoGiveBack ...`
- Classes: `lisaaFamily`, `priyoSelf`, `priyoParent`
- Imports: `lisaaBring`
- Errors: code + stage + location + tip + docs link

## Current limitations

<Callout title="Ecosystem maturity" type="warn">
  PriyoScript core language features are stable, but ecosystem tooling and third-party package
  availability are still growing.
</Callout>

<Callout title="Compatibility scope" type="warn">
  PriyoScript is its own language/runtime, so it does not directly execute JavaScript/Python
  libraries without adapters.
</Callout>

<Callout title="Roadmap reality" type="idea">
  Some planned features may appear first in canary docs before they are stabilized in this handbook.
</Callout>
