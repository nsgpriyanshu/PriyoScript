title: Modules & Packages
description: Organize code with built-in packages and user-defined modules.
---

## Theory

Modules separate code by responsibility. PriyoScript supports:

- Built-in package imports (`lisaaBring math`)
- File-based module imports (`lisaaBring "./module.priyo": alias`)

## Example 1: Built-in package import

```priyo
monalisa {
  lisaaBring math
  priyoTell(math.cube(2))
}
```

## Example 2: Create a user module

`college.priyo`

```priyo
lisaaBox {
  lisaaTask square(x) {
    priyoGiveBack x * x
  }

  priyoKeep campus = "NSEC"

  lisaaShare square
  lisaaShare campus
}
```

## Example 3: Import module as namespace

`main.priyo`

```priyo
monalisa {
  lisaaBring "./college.priyo": college
  priyoTell(college.campus)
  priyoTell(college.square(5))
}
```

## Example 4: Named + aliased imports

```priyo
monalisa {
  lisaaBring "./college.priyo": [campus, square: squareFn]
  priyoTell(campus)
  priyoTell(squareFn(6))
}
```

Comparison:

- JavaScript: `import { x } from './m.js'`
- Python: `from module import x as y`
- Java/C: package systems are compile/link oriented

<Callout title="Module requirement" type="warn">
  File imports must target modules that expose `lisaaBox { ... }`.
</Callout>
