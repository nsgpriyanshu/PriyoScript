## Theory

Modules let you break large programs into small focused files.

Benefits:

- Better organization
- Reuse code across files
- Easier maintenance and testing

## Example 1: Built-in package

```priyo
monalisa {
  lisaaBring math
  priyoTell(math.cube(2))
}
```

## Example 2: Create module

```priyo title="college.priyo"
lisaaBox {
  lisaaTask square(x) {
    priyoGiveBack x * x
  }
  priyoKeep campus = "NSEC"
  lisaaShare square
  lisaaShare campus
}
```

## Example 3: Import module

```priyo title="main.priyo"
monalisa {
  lisaaBring "./college.priyo": college
  priyoTell(college.campus)
  priyoTell(college.square(5))
}
```

## Example 4: Named import alias

```priyo
monalisa {
  lisaaBring "./college.priyo": [campus, square: squareFn]
  priyoTell(squareFn(6))
}
```

## How execution works (simple)

Think of modules like notebooks:

1. `lisaaBox` is notebook boundary
2. `lisaaShare` marks pages visible to others
3. `lisaaBring` opens notebook and imports shared pages

<Callout title="Rule" type="warn">
  File imports must target valid `lisaaBox` module files.
</Callout>
