## Theory

Syntax is the rulebook of a language. If words are correct but order is wrong, parser errors happen.

## Entry block

```priyo
monalisa {
  priyoTell("Program starts")
}
```

## Variables

```priyo
monalisa {
  priyoKeep a = 10
  priyoChange b = 20
  priyoPromise c = 30
}
```

## Literals and expressions

```priyo
monalisa {
  priyoKeep n = 42
  priyoKeep s = "hello"
  priyoKeep ok = priyoTrue
  priyoKeep empty = priyoEmpty
  priyoTell((n + 8) * 2)
}
```

## Comments

```priyo
// single-line
/* multi-line */
```

## Assignment targets

```priyo
monalisa {
  priyoKeep arr = [10, 20]
  arr[0] = 99
}
```

## How execution works (simple)

Parser checks each statement shape first:

- If shape is valid -> compile and run
- If shape is invalid -> stop with syntax error location

Comparison:

- JavaScript: `var/let/const`
- Python: no `const` keyword
- C/Java: typed declarations, stricter upfront type model

<Callout title="Common mistake" type="error">
  Left side of `=` must be assignable (variable/index/property).
</Callout>
