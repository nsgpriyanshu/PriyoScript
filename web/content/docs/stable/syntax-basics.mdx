## Theory

Syntax is the structural grammar that makes source code valid. PriyoScript keeps this grammar small and explicit so beginners can reason about programs quickly.

## 1. Entry block

All executable scripts start with `monalisa { ... }`.

```priyo
monalisa {
  priyoTell("Program starts here")
}
```

## 2. Variables and scope model

```priyo
monalisa {
  priyoKeep a = 10
  priyoChange b = 20
  priyoPromise c = 30
}
```

- `priyoKeep`: function-style mutable binding
- `priyoChange`: block mutable binding
- `priyoPromise`: non-reassignable binding

Comparison:

- C: closest to `int` variables (manual mutability discipline)
- JavaScript: `var` / `let` / `const`
- Python: single binding keyword style (no const keyword)

## 3. Literals and operators

```priyo
monalisa {
  priyoKeep n = 42
  priyoKeep s = "hello"
  priyoKeep b = priyoTrue
  priyoKeep e = priyoEmpty

  priyoTell((n + 8) * 2)
  priyoTell(n >= 10 && b)
}
```

## 4. Comments

```priyo
// single-line

/*
multi-line
*/
```

## 5. Valid assignment targets

```priyo
monalisa {
  priyoKeep arr = [10, 20]
  arr[0] = 99

  priyoKeep obj = priyoCreate Person()
  obj.name = "Priyo"
}
```

<Callout title="Common syntax mistake" type="error">
  Left side of `=` must be assignable (variable, index, or property).
</Callout>
