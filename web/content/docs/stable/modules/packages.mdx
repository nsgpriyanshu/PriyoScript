## 1. Theory

Modular programming means splitting large code into small focused files.  
This improves readability, maintainability, and reuse.  
In PriyoScript, modules are created with `lisaaBox`.  
Exports use `lisaaShare`, and imports use `lisaaBring`.  
Built-in packages (like `math`) are imported directly.  
Local modules are imported by path.  
PriyoScript resolves relative imports from the current file and project-root absolute imports from your project folder.  
It can also auto-resolve `module.priyo` and `module/index.priyo`.  
This structure helps teams work on separate files safely.  
It also reduces copy-paste coding.

## 2. Program Examples

### Example 1: Built-in Package Import

```priyo
monalisa {
  lisaaBring math
  priyoTell(math.cube(2))
}
```

Output:

```text
8
```

### Example 2: Create a Local Module

`college.priyo`:

```priyo
lisaaBox {
  lisaaTask square(x) {
    priyoGiveBack x * x
  }

  priyoKeep campus = "NSEC"

  lisaaShare square
  lisaaShare campus
}
```

Output:

```text
(No direct output; this file defines exports)
```

### Example 3: Import as Namespace

`main.priyo`:

```priyo
monalisa {
  lisaaBring "./college.priyo": college
  priyoTell(college.campus)
  priyoTell(college.square(5))
}
```

Output:

```text
NSEC
25
```

### Example 4: Named + Alias Import

```priyo
monalisa {
  lisaaBring "./college.priyo": [campus, square: sq]
  priyoTell(campus)
  priyoTell(sq(6))
}
```

Output:

```text
NSEC
36
```

### Example 5: Multi-Module Project Pattern

```priyo
monalisa {
  lisaaBring math
  lisaaBring "./college.priyo": college

  priyoTell(math.areaCircle(2))
  priyoTell(college.square(4))
}
```

Output:

```text
12.56636
16
```

### Example 6: Relative Import + Optional `index.priyo`

Directory structure:

```text
project/
  modules/
    billing/
      index.priyo
  main.priyo
```

`modules/billing/index.priyo`:

```priyo
lisaaBox {
  lisaaTask total(base, tax) {
    priyoGiveBack base + tax
  }

  lisaaShare total
}
```

`main.priyo`:

```priyo
monalisa {
  lisaaBring "./modules/billing": billing
  priyoTell(billing.total(100, 18))
}
```

Output:

```text
118
```

### Example 7: Project-Root Absolute Import

```priyo
monalisa {
  lisaaBring "/modules/billing": billing
  priyoTell(billing.total(100, 12))
}
```

Output:

```text
112
```

## 3. Execution Flow

1. Interpreter reads import statements early.
2. Built-in package names resolve from package registry.
3. File path imports resolve with Module v3 rules:
   - `./` and `../` from the importer file
   - `/...` from project root
4. Resolver tries candidates in order:
   - direct path
   - `path.priyo`
   - `path/index.priyo`
5. `lisaaBox` defines module boundary.
6. Only symbols marked by `lisaaShare` are exposed.
7. Import alias maps module exports into current file scope.
8. Program continues normal execution using imported values.

## 4.1 Module v3 Resolution Rules (Quick Reference)

- Built-in package:
  - `lisaaBring math`
- Relative module:
  - `lisaaBring "./utils/pricing"`
  - `lisaaBring "../shared/constants.priyo"`
- Project-root absolute module:
  - `lisaaBring "/modules/billing"`
- Optional auto-fallback:
  - `pricing` -> `pricing.priyo` -> `pricing/index.priyo`

<Callout title="Module Not Found Diagnostics" type="idea">
  If a module is not found, PriyoScript now reports the resolved import path and the exact candidate
  paths it tried, so you can fix the path quickly.
</Callout>

## 5. Comparison With Other Languages

Java:

```java
import java.util.ArrayList;
```

Python:

```python
import math
from module import square as sq
```

Node.js (ESM):

```javascript
import { readFile } from 'node:fs/promises'
```

Node.js (CommonJS):

```javascript
const fs = require('fs')
```

PriyoScript:

```priyo
monalisa {
  lisaaBring math
  lisaaBring "./college.priyo": college
}
```

Difference summary:

- PriyoScript module syntax is explicit and beginner-readable.
- Similar goal to Java/Python/Node imports: reuse code cleanly.
- `lisaaBox` + `lisaaShare` gives clear export boundaries.
