## Theory

Functions are reusable units of behavior. They improve readability, testability, and separation of concerns.

## Syntax

```priyo
lisaaTask name(param1, param2) {
  priyoGiveBack value
}
```

## Example 1: Basic function

```priyo
monalisa {
  lisaaTask add(a, b) {
    priyoGiveBack a + b
  }

  priyoTell(add(10, 5))
}
```

## Example 2: Recursion

```priyo
monalisa {
  lisaaTask factorial(n) {
    prakritiIf (n <= 1) {
      priyoGiveBack 1
    }
    priyoGiveBack n * factorial(n - 1)
  }

  priyoTell(factorial(5))
}
```

## Example 3: Closure

```priyo
monalisa {
  lisaaTask makeCounter() {
    priyoChange count = 0

    lisaaTask next() {
      count = count + 1
      priyoGiveBack count
    }

    priyoGiveBack next
  }

  priyoKeep counter = makeCounter()
  priyoTell(counter())
  priyoTell(counter())
}
```

## Example 4: Function as reusable business rule

```priyo
monalisa {
  lisaaTask isAdult(age) {
    priyoGiveBack age >= 18
  }

  priyoTell(isAdult(21))
}
```

Comparison:

- C: function pointers are explicit and low-level
- Java: methods inside classes are dominant
- JavaScript/Python: first-class functions and closures (similar mental model)

<Callout title="Important" type="warn">
  `priyoGiveBack` is valid only inside a function body.
</Callout>
